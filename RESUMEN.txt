┌─────────────────────────────────────────────────────────────────┐
│  🚀 INICIO RÁPIDO                                               │
└─────────────────────────────────────────────────────────────────┘

1️⃣  PERSONALIZAR DATOS
   → Edita: data.js
   → Cambia: nombre, fecha, lugar, invitados, itinerario

2️⃣  GENERAR PÁGINAS INDIVIDUALES
   $ node generate.js

3️⃣  PROBAR LOCALMENTE
   $ python3 -m http.server 8000
   → Abre: http://localhost:8000

4️⃣  VER VISTA PREVIA
   → Abre: http://localhost:8000/preview.html

5️⃣  DESPLEGAR
   $ ./deploy.sh
   O copiar archivos a tu servidor web

┌─────────────────────────────────────────────────────────────────┐
│  🔐 URLS PERSONALIZADAS                                         │
└─────────────────────────────────────────────────────────────────┘

Cada invitado recibe una URL única:

Formato: https://tudominio.com/[UUID].html

Ejemplo:
https://tudominio.com/a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d.html

La invitación detecta automáticamente el UUID en la URL
y muestra la información personalizada del invitado.

┌─────────────────────────────────────────────────────────────────┐
│  📦 DESPLIEGUE EN PRODUCCIÓN                                    │
└─────────────────────────────────────────────────────────────────┘

OPCIÓN 1: Nginx (Recomendado)
   1. Copiar archivos: sudo cp -r * /var/www/html/invitacion/
   2. Configurar nginx.conf
   3. Reiniciar: sudo systemctl restart nginx

OPCIÓN 2: Apache
   1. Copiar archivos: sudo cp -r * /var/www/html/invitacion/
   2. Configurar .htaccess (si es necesario)
   3. Reiniciar: sudo systemctl restart apache2

OPCIÓN 3: Hosting estático (GitHub Pages, Netlify, Vercel)
   1. Subir archivos al repositorio
   2. Activar despliegue automático
   3. Configurar dominio personalizado

OPCIÓN 4: CDN (Cloudflare Pages, AWS S3 + CloudFront)
   1. Subir archivos al storage
   2. Configurar distribución
   3. Añadir dominio personalizado
